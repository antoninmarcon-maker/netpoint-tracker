[2026-03-19] | Set winner logic used `>=` which silently awards ties to blue team | Always use strict `>` for winner comparison; prevent action when scores are equal
[2026-03-19] | SW cache filter had two `.filter()` calls where the second negated the first | Test filter chains by tracing with a concrete example before deploying
[2026-03-19] | Nominatim API called without User-Agent header (violates ToS) | Always add User-Agent to third-party API calls; centralize API calls to avoid repeating the fix in 3 files
[2026-03-19] | Heatmap normalization looked like a sidesSwapped bug but was actually correct | Verify assumptions by reading the full data flow before filing a bug
[2026-03-19] | analyze-match Edge Function seemed to have auth bypass but actually takes client data, not DB IDs | Check if function queries DB before flagging auth issues
[2026-03-19] | Theme flash caused by applying dark class in React instead of before first paint | Always apply theme in a blocking `<script>` in `<head>` before CSS loads
[2026-03-24] | Bug analysis in todo.md was wrong — `hasRating: false` vs `undefined` misdiagnosed as "ratings never requested" when matchRules OR logic saved it | Always trace the FULL code path before documenting root cause; check OR/fallback conditions
[2026-03-24] | Custom action properties (hasRating) lost in getVisibleActions mapping — worked by accident via fallback | When mapping object types, include ALL relevant fields; don't rely on downstream fallbacks
[2026-03-24] | Zod schemas stripped `rating` field from Points/RallyActions — Zod strips unknown fields by default with `.object()` | When adding a field to a TypeScript interface, ALWAYS update the corresponding Zod schema; audit all schemas when adding persistent fields
[2026-03-24] | Pending spots invisible — Supabase default limit is 1000 rows, DB had 3035 spots, query without filter returned only validated ones | Always filter server-side with .eq() instead of relying on client-side filtering when dataset exceeds 1000 rows; never omit a status filter expecting to get "all" rows
[2026-03-24] | Renamed MODERATOR_EMAIL to MODERATOR_EMAILS but missed one reference in onAuthStateChange callback — caused ReferenceError in production | When renaming a variable, grep the ENTIRE codebase for all occurrences before committing; don't trust replace_all on partial matches
[2026-03-24] | Supabase DELETE with RLS returns { error: null, count: 0 } when policy blocks — no error thrown | Always use `{ count: 'exact' }` and check `count === 0` to detect silent RLS rejections; never assume no-error means success
[2026-03-24] | Manual cascade delete (comments → photos → spot) failed silently because RLS owner policies blocked deleting other users' comments | Rely on FK ON DELETE CASCADE at DB level instead of manual cascade from client; add moderator-specific RLS policies for admin operations
[2026-03-24] | Hardcoded `rgba(9,9,11,...)` backgrounds break light mode — Supabase-style dark glass effects aren't theme-aware | Create CSS utility classes (glass-overlay, glass-btn) with light/dark variants instead of inline hardcoded rgba values
[2026-03-24] | Using `key={refreshKey}` on a map component destroys and recreates the entire Leaflet instance, resetting zoom/position | Never use React key to trigger data refresh on stateful components like maps; pass a refreshTrigger prop instead
[2026-03-24] | Map data re-fetched on every pan/zoom because useEffect depended on bounds state object | Use a ref for bounds + a version counter; only bump the counter when crossing meaningful thresholds (zoom band change, or panning while zoomed in)
[2026-03-24] | Theme reverts to dark on reload because blocking script reads raw `localStorage.theme` but useTheme writes to namespaced `theme_<userId>` key | When a blocking `<script>` in `<head>` reads a localStorage key, ensure the runtime code also writes to that same raw key — not just a namespaced variant
[2026-03-26] | V1 enrichment used 200m radius → photos from unrelated places 300km away | Always use concentric search with confidence scoring; reject low-confidence matches
[2026-03-26] | V1 stored Google Places photo URLs directly → depend on API key, may expire | Always download photos to Supabase Storage for permanence
[2026-03-26] | V1 generated descriptions without verifiable sources → hallucinations | Never invent info in descriptions; only state facts from named sources
[2026-03-26] | V1 agents shared context across spots → cross-contamination of data | Fresh agent per spot to prevent hallucination from prior context
[2026-03-26] | Map search bar not clickable — Leaflet captured touch/click events on overlay, and a full-width `pointer-events-auto` back button div covered the input | Always call `L.DomEvent.disableClickPropagation` on Leaflet overlays; never use full-width pointer-events-auto on an overlay if only one small element needs it
[2026-03-26] | Search returned 0 results for "Aire multisports Bellême" — Nominatim is a geocoder (locations only), and Supabase `ilike '%full query%'` requires exact substring match | Split multi-word queries into individual words with OR matching, then rank results by number of matched words; never assume a single ilike will work for multi-word queries with location context
[2026-03-26] | Supabase OR query limit=8 missed relevant results because common substrings like "aire" flooded the top results | When using OR+ranking, fetch a larger batch (30+) from DB then sort/slice client-side; small limits defeat relevance ranking
